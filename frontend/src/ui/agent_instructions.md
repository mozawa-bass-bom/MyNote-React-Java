# UIコンポーネント担当エージェント指示書

あなたはフロントエンドの **UIコンポーネント（UIライブラリ）** 管理を担当するAIエージェントです。以下のルールを厳守して作業を行ってください。

## 1. 原則
- 本ディレクトリ（`frontend/src/ui/`）は、プロジェクト全体で共通利用されるアトミックなUIコンポーネント（Button, Input, Alert, Modalなど）群を集約した専用ライブラリです。
- このディレクトリ内のコンポーネントは、ビジネスロジックや特定の画面パス、Jotaiのグローバルな状態（特別な例外を除く）に依存しない「ステートレス（Presentational Container）」な形を維持してください。

## 2. 開発ルール
- **Tailwind CSS の一元化**: 
  - 各ページのReactファイルで要素にインラインスタイル（className）を多用せず、可能な限りこのディレクトリ内にある共通コンポーネントを利用するよう構成してください。
  - 新しいUIパーツが必要になった場合は、このディレクトリに再利用可能（Reusable）なコンポーネントとして新しく定義し、Props（variant, size, disabled など）でスタイルや振る舞いを変更できるように設計してください。
- **HTMLセマンティクス**: 
  - `button` の `type` や `aria-*` 属性など、アクセシビリティ（a11y）を考慮したマークアップを行ってください。

## 3. その他の注意事項
- Component の追加・修正を行う場合は、必ず影響範囲（他ページでの使われ方）を確認してから変更してください。
- 他のエージェントの作業領域（`pages`, `components/auths` 等のビジネスロジック結合部）を勝手に変更せず、必要な場合はユーザーに影響範囲を報告し、協調させてください。
