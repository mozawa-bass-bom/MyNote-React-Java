# 授業ノートアプリ要件定義書

第1.0版
作成者: 小澤 基良

## 1. 本書の位置付け
本書は、会議・授業参加時におけるAIを利用した事前準備アプリ（以下、「授業ノートアプリ」）に関する要件を記す。

## 2. システム導入の背景
語学学習や会議参加時には、音声内容を聞き取りながらノートを取る必要があるが、これには高い集中力とマルチタスク能力が求められる。内容の聞き漏らしや記録漏れが発生しやすく、学習効率や会議の理解度に支障をきたすケースが多い。特に事前準備が不十分な状態では負担が大きくなる。

## 3. システム導入の目的
PDFを読み込むことで必要であろう注釈や解説をAIで生成しあらかじめ表示することで、学習効率の向上や記入漏れの減少、話の理解への集中力の増加を目指す。事前準備の手間を効率化し、負担を軽減する。
【PDF読み込みによる表示内容】
- 読み込んだPDFの概要と目次
- 目次毎の解説
- ページ毎の画像・注釈
- フォルダ階層のタブメニュー

※今後の課題：著作権などのコンプライアンスチェック

## 4. システムの概要
### 4.1 システム化の範囲
PDFの読み込み、AIでの解説生成、ユーザー毎の画像・テキストの管理。
### 4.2 ユーザーの種類と機能
- 利用者：PCやスマホからアクセスし、PDFアップロード、画像・AI注釈の閲覧および手動編集・削除などを行う。
- 管理者(Admin)：全ユーザーの対象データを管理、ユーザー登録・削除を行う。

## 5. 機能要件 (機能一覧)
- **F01-1 ユーザー登録機能**: メール・パスワードによる登録、パスワードリセット
- **F01-2 ユーザー認証機能**: ログイン・ログアウトのセッション管理
- **F02 ユーザー削除機能 (Admin)**: 管理者によるアカウント削除 (MySQL Cascade Delete, Firebase画像削除)
- **F03-1 PDFアップロード機能**: PDFアップロード、ページ分割、画像化・Firebase保存、OCR処理
- **F03-2 AI注釈生成機能**: テキストを元にAI解説を生成 (JSONで受け取る)
- **F03-3 JSONパース機能**: JSONをDTOに変換し、画像情報とセットでMySQLに保存、HTML出力
- **F04 階層カテゴリ管理機能**: 2階層のカテゴリ(フォルダ)の作成・編集・削除
- **F05-1 ノート閲覧機能**: 画像・テキストの参照
- **F05-2 ノート編集機能**: ユーザー手動での追加・編集
- **F05-3 ファイル・ページ削除機能**: ページ単位の画像・テキスト削除
- **F06 プロンプト追加機能**: 解説の内容や口調の独自プロンプト登録
- **F07 アクセス権管理機能**: 自身がアップロードしたファイルのみ閲覧・編集可

## 6. 非機能要件
- **パフォーマンス**: AI生成・画像保存の非同期処理、JSでの処理状況表示
- **セキュリティ**: パスワードのハッシュ化、HTTPS、適切なアクセス権管理
- **拡張性**: 検索機能の実装検討
- **保守性**: ソースコードのモジュール化、ドキュメントの充実
- **ユーザビリティ**: シンプルUI、レスポンシブ対応(PC・スマホ)
- **ログ管理**: 重要操作のログ記録
- **可用性**: エラー/例外時のシステム安定化(例外処理)
